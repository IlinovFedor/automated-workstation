\Conclusion

В ходе выполнения творческой работы разработано автоматизированное рабочее место диспетчера по расписанию. Поставленная цель достигнута, все задачи решены.

Проведён анализ предметной области, спроектирована структура реляционной базы данных, разработана OpenAPI-спецификация REST API. Реализован сервер на Go с автогенерацией кода по спецификации. Разработан десктопный клиент на C++/Qt6 с автогенерацией API-клиента. Реализован импорт расписания из XLSX-файлов и экспорт в формат iCalendar. Выполнена контейнеризация приложения средствами Docker~\cite{docker:doc}.

Проект имеет перспективы развития. В будущем возможно добавление автоматического составления расписания в зависимости от заданных параметров: рабочие дни преподавателей, учебное время групп, учёт санитарно-эпидемиологических правил и норм. Также возможна реализация формирования печатного варианта текущей схемы расписания.

Кроме того, потенциальным направлением развития является создание веб-версии приложения. В качестве примера подобной реализации в каталоге frontend представлен демонстрационный веб-интерфейс, включённый в конфигурацию Docker Compose. Данный интерфейс был полностью сгенерирован с использованием нейросетевых инструментов и не относится к основной части выполненной работы, выступая исключительно в качестве демонстрационного прототипа.

Указанный веб-клиент использовался автором работы и рядом других пользователей (в частности, однокурсниками) в течение учебного года в процессе эксплуатации системы. Практическое применение данного интерфейса подтвердило удобство и корректность выбранной архитектуры приложения, основанной на едином API и централизованной серверной логике.

Полученные навыки проектирования архитектуры клиент-серверных приложений, работы с OpenAPI, Go, C++/Qt, реляционными базами данных и Docker имеют практическую ценность и могут быть применены при разработке более сложных программных систем.
